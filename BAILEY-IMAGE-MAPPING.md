## Image Naming Convention

Our established convention (from `LESSON-SYSTEM-ARCHITECTURE.md`):

- **Filename**: `{session-name}.png` — must match the `session_name` field in the sessions table
- **Storage path**: `/media/pitch-diagrams/{age-group}/{skill}/{session-name}.png`
- **Full URL**: `https://{project}.supabase.co/storage/v1/object/public/lesson-media/media/pitch-diagrams/{age-group}/{skill}/{session-name}.png`

For Academy lessons, the age-group folder will be `junior` (since team_type = Junior Football = U9-U12).

### Handling Reused Images

Bailey reuses the same drill diagram across many lessons (e.g. `image5.png` is the standard warmup on 28 slides). In our system, each session has its own `diagram_url`, so:

- **Option A**: Copy the same image file with different names for each session. Simple but duplicates storage.
- **Option B**: Upload once with a generic name and have multiple sessions point to the same URL. Saves storage but breaks the `{session-name}.png` convention.
- **Recommended: Option A** — copy with session-specific names. Storage is cheap, and it keeps the naming convention clean. If Bailey later provides unique images per session, we just replace the file.

### Rename Script

A rename/copy script will be generated once all lesson SQL migrations are created (so all session names are known). The script will:
1. Read `bailey-image-mapping.json`
2. For each slide/session, copy `media/imageN.png` to `{session-name}.png`
3. Organise into `/media/pitch-diagrams/{age-group}/{skill}/` folders
4. Output a SQL script to update `diagram_url` for all sessions
