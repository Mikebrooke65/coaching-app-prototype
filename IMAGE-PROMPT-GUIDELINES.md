# Image Prompt Guidelines for Pitch Diagrams
## Standard Template for AI Image Generation

---

## Purpose
This document defines the mandatory structure for all pitch diagram image generation prompts to ensure consistency, clarity, and quality across all 32 lessons (128 sessions).

---

## Standard Prompt Template

Every pitch diagram prompt MUST follow this exact structure:

```
Create a football (soccer) coaching pitch diagram from top-down view. 

[PITCH SETUP - dimensions and markings]

[PLAYERS - positions, bibs, and ball possession]

[MOVEMENT - arrows and patterns]

[ADDITIONAL ELEMENTS - indicators, labels, legends]

Use consistent icons, colors (orange/blue), and arrow styles. High contrast, clean minimal design. White background. 4:5 aspect ratio. Minimum 800px on short edge. PNG format. Professional coaching diagram style for mobile app display.
```

---

## Mandatory Components

### 1. Opening Statement (ALWAYS INCLUDE)
```
Create a football (soccer) coaching pitch diagram from top-down view.
```

**Why:** Clarifies this is soccer/football, not American football or other sports.

---

### 2. Pitch Setup Section
**Format:**
```
Show [dimensions] [pitch type/area]. [Markings/equipment description].
```

**Examples:**
- "Show an 18x18 meter square marked with orange cones at corners."
- "Show five 9x9 meter grids arranged in a row."
- "Show three 14x9 meter channels side by side, each with small goals at both ends."
- "Show 27x18 meter pitch with small goals at each end."

**Rules:**
- ALWAYS use metric measurements (meters, not yards)
- Specify exact dimensions
- Describe all markings (cones, goals, lines)
- Use "meter" not "m" for clarity

---

### 3. Players Section
**Format:**
```
Display [number] [role] wearing [color] bibs [with/without ball]. [Additional player details].
```

**CRITICAL RULES:**
1. **ALWAYS specify who has the ball**
2. **ALWAYS use "football (soccer ball)" not just "ball"**
3. **ALWAYS use orange for attackers, blue for defenders**
4. **ALWAYS specify bib colors**

**Examples:**
- "Display 3 pairs of players: attackers wearing orange bibs with football (soccer balls), defenders wearing blue bibs in side-on jockey stance 2-3 meters away."
- "Each grid contains one orange player (attacker) with football (soccer ball) and one blue player (defender)."
- "Orange attacker dribbling forward with football (soccer ball), blue defender jockeying backward in side-on stance."
- "Display 4v4 game: 4 orange players vs 4 blue players. Orange player in center has football (soccer ball)."

**Color Standards:**
- Orange bibs = Attackers/players with ball
- Blue bibs = Defenders/players without ball
- Specify which specific player has the ball

---

### 4. Movement Section
**Format:**
```
Use [arrow type] to show [movement description]. [Additional movement patterns].
```

**Arrow Types:**
- Solid arrows = actual movement (player running, ball path)
- Dashed arrows = alternative movement or return path
- Dotted arrows = potential/optional movement
- Curved arrows = change of direction

**Examples:**
- "Use curved arrows to show attacker movement patterns and defender mirroring movements."
- "Use solid arrows for attacker's forward movement, dashed arrows for defender's backward jockey, then solid arrow for forward tackle."
- "Use arrows to show ball movement and defensive pressure."

**Rules:**
- Specify arrow type (solid, dashed, dotted, curved)
- Describe what each arrow represents
- Keep movement patterns clear and simple

---

### 5. Additional Elements Section
**Format:**
```
Include [element type] [description]. [Additional indicators].
```

**Common Elements:**
- Legends (player icons, arrow meanings)
- Indicators ("heavy touch", "bonus point", "X" for incorrect)
- Insets (close-up details, foot positioning)
- Labels (zone names, distances)

**Examples:**
- "Include simple legend with player icons and arrow meanings."
- "Show incorrect head-on approach with dotted line and X mark."
- "Include small inset diagram showing detailed foot positioning."
- "Add 'heavy touch' indicator icon."
- "Include 'bonus point' star icon near successful tackle area."

---

### 6. Technical Specifications (ALWAYS INCLUDE)
```
Use consistent icons, colors (orange/blue), and arrow styles. High contrast, clean minimal design. White background. 4:5 aspect ratio. Minimum 800px on short edge. PNG format. Professional coaching diagram style for mobile app display.
```

**This closing statement is MANDATORY for every prompt.**

## Checklist for Every Prompt

Before generating an image, verify your prompt includes:

- [ ] Opening: "Create a football (soccer) coaching pitch diagram from top-down view"
- [ ] Pitch dimensions in METERS
- [ ] Player count and positions
- [ ] Bib colors specified (orange/blue)
- [ ] WHO HAS THE BALL clearly stated
- [ ] "football (soccer ball)" terminology used
- [ ] Movement arrows described (type and meaning)
- [ ] Any special indicators or legends
- [ ] Closing technical specifications paragraph

---

## Common Mistakes to Avoid

❌ **DON'T:**
- Use "yards" (use meters)
- Say "ball" (say "football (soccer ball)")
- Forget to specify who has the ball
- Mix up colors (orange = attackers, blue = defenders)
- Omit the technical specifications closing
- Use "American football" or ambiguous "football"

✅ **DO:**
- Use metric measurements
- Specify "football (soccer ball)"
- Always state which player has possession
- Use consistent orange/blue color scheme
- Include all 6 mandatory sections
- Be specific about player positions and movements

## Quality Standards

Every generated image must meet these criteria:

**Visual Quality:**
- Clean, uncluttered design
- High contrast for mobile viewing
- Consistent icon style across all images
- Professional coaching diagram appearance

**Technical Quality:**
- PNG format
- 4:5 aspect ratio (800x1000px minimum)
- White background
- Minimum 800px on short edge

**Content Quality:**
- Accurate representation of drill/exercise
- Clear indication of ball possession
- Logical movement patterns
- Appropriate level of detail

---

## Usage Workflow

1. **Read the session content** (organisation, pitch layout description)
2. **Identify key elements:**
   - Pitch dimensions
   - Number of players
   - Who has the ball
   - Movement patterns
   - Special indicators needed
3. **Build prompt using template:**
   - Opening statement
   - Pitch setup
   - Players (with ball possession)
   - Movement
   - Additional elements
   - Technical specifications
4. **Review against checklist**
5. **Generate image**
6. **Verify quality standards**
