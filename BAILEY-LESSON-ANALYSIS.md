## Step 7: Database Schema Changes Required

Before we can import Academy lessons, the schema needs two additions:

### 1. Add `division` field to lessons table
- Values: `Community`, `Academy`
- Our existing 16 U9 lessons = Community
- Bailey's lessons = Academy
- Required migration: `ALTER TABLE lessons ADD COLUMN division TEXT DEFAULT 'Community';`

### 2. Add `team_type` field to lessons table (instead of changing age_group)
- Values: `First Kicks`, `Fun Football`, `Junior Football`, `Youth Football`, `Senior`
- These map to age group ranges:
  - First Kicks = U4–U6
  - Fun Football = U7–U8
  - Junior Football = U9–U12
  - Youth Football = U13–U17
  - Senior = Adult
- This is the correct tagging level — a lesson tagged "Junior Football" covers U9–U12
- Existing `age_group` field can remain for backward compatibility or be phased out
- Bailey's lessons = `Junior Football` team type
- Our existing U9 lessons = could be `Junior Football` or keep specific `U9` age_group
- Required migration: `ALTER TABLE lessons ADD COLUMN team_type TEXT;`
- Same field may be useful on `sessions` table too

### 3. Potentially expand `skill_category` CHECK constraint
- Current allowed values: Tackling, Marking, Pressing, Intercepting, Dribbling, Ball Striking, 1v1, Passing/Receiving
- Bailey's lessons include skills not in this list: Shielding, Possession/Rondo, Crossing & Finishing, Defending Transition, Ways to Break the Line
- Options:
  - Add new categories to the CHECK constraint
  - Use a more flexible approach (remove CHECK, use a reference table)
  - Wait for Bailey to confirm skill mappings before changing
- **Decision: Wait for Bailey** — park this until he reviews
